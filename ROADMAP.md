## 🗺️ 学习路线

### 阶段一：C++ 基础巩固

<!-- 优先级：高 | 预计周期：2-3 周 -->

- [ ] 指针与引用 — 理解内存模型，区分栈与堆
- [ ] 动态内存管理 — `new` / `delete`，认识内存泄漏
- [ ] 面向对象深入 — 继承、多态、虚函数、抽象类
- [ ] 运算符重载 — 让自定义类型像内置类型一样使用
- [ ] 异常处理 — `try` / `catch` / `throw`，编写健壮代码
- [ ] 模板基础 — 函数模板与类模板，理解泛型编程

**实践目标**：扩展现有 TaskManager，加入继承体系（如 `ScheduledTask`、`RecurringTask`）

---

### 阶段二：STL 与现代 C++

<!-- 优先级：高 | 预计周期：3-4 周 -->

- [ ] STL 容器全览 — `map` / `set` / `deque` / `list` / `unordered_map`
- [ ] STL 算法 — `<algorithm>` 常用算法：`sort` / `transform` / `accumulate`
- [ ] 迭代器 — 理解迭代器分类与自定义迭代器
- [ ] 智能指针 — `unique_ptr` / `shared_ptr` / `weak_ptr`，告别手动 `delete`
- [ ] 右值引用与移动语义 — `std::move`，理解性能优化
- [ ] C++11/14/17 新特性 — `auto` / `range-for` / `std::optional` / `std::variant`

**实践目标**：用 `shared_ptr` 重构 TaskManager，支持任务持久化到文件

---

### 阶段三：工程化与设计

<!-- 优先级：中 | 预计周期：3-4 周 -->

- [ ] 项目构建 — CMake 入门，管理多文件项目
- [ ] 设计模式（C++ 实现）— 单例、工厂、观察者、策略模式
- [ ] 单元测试 — Google Test 框架，编写可测试的代码
- [ ] 文件 I/O — `std::fstream`，实现数据持久化
- [ ] 多线程基础 — `std::thread` / `std::mutex` / `std::async`
- [ ] 代码规范 — 命名约定、头文件守卫、`const` 正确性

**实践目标**：将 TaskManager 拆分为多文件项目，用 CMake 构建，加入 GTest 测试

---

### 阶段四：进阶与实战

<!-- 优先级：中 | 预计周期：4-6 周 -->

- [ ] 网络编程 — Socket 基础，实现简单的客户端/服务端
- [ ] 数据结构实现 — 手写链表、二叉树、哈希表
- [ ] 算法实践 — 排序、搜索、动态规划（配合 LeetCode）
- [ ] 跨平台开发 — 处理 Windows / Linux 差异
- [ ] 性能分析 — `valgrind` / 性能剖析工具，定位瓶颈
- [ ] 开源项目阅读 — 阅读小型开源项目源码，学习工程写法

**实践目标**：实现一个命令行 TODO 应用，支持文件存储、多线程刷新、网络同步

## 💡 学习原则

1. **动手为主** — 每学一个知识点，必须写代码验证
2. **渐进式** — 不跳跃，基础不牢不进入下一阶段
3. **问题驱动** — 遇到问题先独立思考，再查资料
4. **定期回顾** — 每完成一个阶段，回头重写之前的代码
5. **记录总结** — 在 `notes/` 目录下记录学习心得与踩坑经验
